Scenario
Maya is preparing for a quarterly business review.

She has three months of account notes, call summaries, and email threads from a key client. She needs to distill all of it into a two-page executive summary — and she has 90 minutes before the meeting.

The account notes contain client names and deal values but no personal data or confidential contracts.

M
Maya
Client Success Manager, NTAM
Knowledge Check — Question 1 of 3
Which of the following is the BEST way for Maya to use the AI tool in this situation?
A
Paste all of the account notes directly into the tool and ask it to write the QBR summary for her, then submit it without reviewing.
B
Use the tool to draft a structured summary from the account notes, then review the output carefully and edit it before the meeting.
C
Skip using the tool because the account notes contain client names, which makes this an inappropriate use case.
D
Ask the tool to browse the client's public website and add that information to the QBR summary.
Correct. Using the tool to draft and then reviewing the output is exactly right. The account notes don't contain personal data or confidential contracts, so this is an appropriate use. Maya's judgment — not the tool — is the final check before submission.
Not quite. Review the acceptable use guidelines from slide 4. The key principle: the tool can help draft, but you always review before submitting. Option B describes this correctly. Try again.

Design Rationale — Why This Slide Is Built This Way
Instructional Design Decisions
Slide 05 · Scenario + MCQ
Design Choices
Scenario framingGrounds the question in a realistic work situation rather than abstract recall. Forces application of knowledge, not just recognition.
Named characterMaya is a CS persona — intentional. Learners in CS roles see themselves. Other functions still engage because the situation is universal.
Split layoutScenario context stays visible while the learner answers. Eliminates the need to scroll back, reducing cognitive load.
Four distractorsEach wrong answer targets a specific misconception: over-reliance on the tool (A), data anxiety misapplied (C), and capability confusion (D).
Technical Decisions
Branched feedbackCorrect and incorrect feedback are specific, not generic. Wrong answer feedback references the exact slide where the answer is found.
Next lockedLearner cannot advance until an answer is selected. Prevents click-through without engagement.
Avatar offAvatar narration paused on interaction slides to avoid competing attention. VO introduces the slide before interaction begins.
Bloom's L3This question targets Apply, not just Remember. The scenario requires learners to evaluate a situation against principles, not recall a definition.
